## Design Context

### Users
This product is used by operations and platform engineers managing private Nginx deployments. They work in a control-console context where repeated inspection, editing, publishing, and auditing need to be fast and clear.

### Brand Personality
The interface should feel precise, calm, and operational. It should communicate confidence and traceability rather than marketing energy.

### Aesthetic Direction
Use a refined operations-console style: light, dense, table-friendly layouts with clear forms, compact status badges, practical filters, and restrained use of the existing Nginx control green theme. Dark mode must remain supported.

### Design Principles
- Prioritize scanability and repeated action over decorative composition.
- Keep page structure modular: list pages, edit pages, and services should be split by feature.
- Use compact panels, tables, filters, and explicit statuses for operational clarity.
- Preserve existing Angular, NG-ZORRO, ng-alain, Less, and theme-token patterns.
- Make risky actions explicit with confirmation and clear feedback.
